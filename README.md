# GEO Assistant

GEO Assistant 是一个本地可运行的 GEO 分析和写作辅助系统，内置从品牌定义、Query定义、GEO分析、内容创作完整链路。

## GEO优化流程的定义

### 1.定义品牌

- 梳理品牌，定义品牌和市场关系：包含品牌名称、别名、品牌地址（可以是官网，也可以是发布贴文的可访问地址）、竞品名称和地址；
- 定义品牌资产：围绕品牌卖点，定义希望AI表达的卖点列表和核心数据（证据模块），用于检测和后续内容创作时精准匹配，便于有针对性地分析和创作。

### 2.数据准备

- 深刻理解用户场景和意图类型，按照问题场景定义Query集，按照问题意图（从模糊到精确）为每个Query集定义Query，Query模拟用户可能的提问；
- 基于Query集发起平台采样、引用链接采集，提取引用内容中品牌相关的上下文，为分析提供数据支撑。

### 3.GEO分析

- 基础分析：可按照不同批次、Query集、Query意图进行查看，包含答案中的品牌位次、竞品位次、相对排序，引用中品牌域名的命中率，以及TOP引用来源；
- 引用分析：分析最新批次引用中对品牌和竞品的描述，总结出品牌优势和劣势、竞品的优势和劣势；
- 答案分析：分析最新批次答案中对品牌的描述，以及对品牌描述和定义的品牌资产的契合度；
- 核心数据：同一批次下或跨批次差异和稳定性：不同平台品牌/竞品位次、品牌地址命中率、引用来源TOP10、品牌资产命中率。

### 4.内容梳理和创作

- 通过引用查询等功能，查询、分析不同平台高引用地址来源，尤其是提及品牌或竞品的高引用来源，一方面定位有效的发布渠道，另一方面分析其提到的优缺点和品牌定义的差异；
- 通过内容资产功能，完善品牌资产的证据模块；
- 通过内容创作功能，可以选择已定义的内容资产证据模块、引用的品牌/竞品优缺点快速行文、修改，形成定义全面的GEO优化的内容，发布到有效的发布渠道，以期在互联网上阐明品牌定义。

### 5.数据跟踪

- 内容发布后，每隔7-15天针对Query集进行重采样，对比批次之间的数据差异，跟踪效果；
- 根据效果决定下一步优化的方向。



## 本地初始化

```bash
npm install
npm run db:setup
npm run seed
```

可选：生成演示采样和指标数据。

```bash
npm run demo
```

启动开发服务：

```bash
npm run dev
```

默认访问：

```text
http://localhost:3000
```

## 生产部署

构建：

```bash
npm run build
```

启动默认端口：

```bash
npm run start
```

指定端口启动，例如 8500：

```bash
npm run start -- -p 8500
```

后台运行示例：

```bash
nohup ./node_modules/.bin/next start -p 8500 > /tmp/geo-assistant-8500.log 2>&1 < /dev/null &
```

验证：

```bash
curl -I http://localhost:8500/
```
